BR-1 · Split Hr / Min Entry
NEW Time is entered as two separate fields: Hr (1–12) and Min (0–59). Each field supports both direct typing and dropdown selection.
Typing 29 in the Min field is accepted exactly — it's not forced to a 5-minute boundary. The green badge and confirmation confirm the exact value is saved.
Before
Single dropdown
Must pick preset. 10:29 AM unavailable.
After
Hr + Min inputs
Type 10 in Hr, 29 in Min → saved exactly.
Try it: Type 10 in Hr, then 29 in Min. Watch the green exact-time confirmation appear.
BR-4 · Google Calendar Timed Event
Any appointment saved with a specific Hr and Min syncs to Google Calendar as a timed event, not an all-day event.
The all-day event bug was a downstream consequence of the time not resolving. With split fields, the value is always unambiguous.
⚠️ Comms Impact — QA Required
The SMS/email token {appointment-time} resolves from the stored time. QA must confirm that 10:29 AM (not 7:00 AM) appears in outbound notifications.

BR-3 · No Default Override
FIXED The placeholder -- only appears before the user has interacted with each field. Once a value is entered, the system preserves it through save. 7:00 AM never reasserts itself.
BR-5 · Timezone Integrity
Times entered are stored and synced in the company's configured timezone. The UI passes the composed value through faithfully.
Edge Case · Invalid Entry
EDGE CASE If a user types an out-of-range Hr (e.g. 99) or Min (e.g. 99), the offending field shows a red invalid border and the Schedule button is blocked.
The error surfaces on blur and persists until a valid value is entered. It does not fire while the user is still typing.
Try it: The Hr field is pre-filled with 99 (invalid). Click Schedule — it is blocked. Fix it to 10 and try again.

Edge Case · Clear Field & Save
EDGE CASE If Hr or Min is empty and the user attempts to save, 7:00 AM is not silently applied.
Instead the modal shows a warning that the appointment will sync as an all-day Google Calendar event, giving the user a chance to add a time before confirming.

BR-6 · All Appointment Types
The split Hr/Min time field is the shared component used across all appointment types: on-site estimates, job appointments, warranty work, and any others. All inherit the same behavior.
